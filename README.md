# AROMA CARE - שלד הפרויקט

מסמך זה מסביר את מבנה הפרויקט בשפה פשוטה, גם אם אינך מפתח/ת.

## מה יש כאן כרגע

זהו **שלד** בלבד - התיקיות והקבצים הבסיסיים שעליהם נבנה את שאר האתר. עדיין:
- אין עיצוב סופי מלא (רק דוגמת קונספט)
- אין חיבור אמיתי ל-Wix (יש נתוני דמו מהאקסל)
- אין עמודי קטגוריה/מוצר מלאים (יגיעו בשלבים הבאים)

## מבנה התיקיות - מה כל אחת עושה

| תיקייה | מה יש בה |
|---|---|
| `app/` | הדפים עצמם של האתר (עמוד בית, וכו') |
| `components/ui` | רכיבים כלליים לשימוש חוזר (כפתורים, שדות) |
| `components/layout` | ראש עמוד, תחתית עמוד, ניווט |
| `components/product` | כל מה שקשור לתצוגת מוצר |
| `components/cart` | סל הקניות |
| `components/collections` | עמודי קטגוריה/קולקציה |
| `lib/wix` | קוד החיבור ל-Wix (עדיין לא פעיל בפועל) |
| `lib/analytics` | מעקב אירועים (Google Analytics וכו') |
| `lib/seo` | הגדרות קידום אתרים (כותרות, תיאורים) |
| `types/` | "טפסים" שמגדירים איך נראה מוצר, עגלה וכו' |
| `hooks/` | לוגיקה לשימוש חוזר (כרגע: ניהול עגלה) |
| `services/` | השכבה שמביאה נתוני מוצרים - היחידה שמדברת עם המקור (דמו או Wix) |
| `mocks/products.json` | **21 המוצרים שכבר ניתחנו, עם הנתונים האמיתיים מהאקסל** |
| `config/` | הגדרות מרכזיות, כולל המתג בין מצב דמו למצב Wix |
| `scripts/` | כלים להרצה חד-פעמית (כמו ייבוא מוצרים) |
| `public/` | קבצים סטטיים כמו תמונות |

## המתג החשוב ביותר: MOCK MODE מול WIX LIVE MODE

בקובץ `.env` (שייווצר מ-`.env.example`) יש שורה:

```
NEXT_PUBLIC_DATA_MODE=mock
```

- כרגע זה `mock` - האתר קורא את המוצרים מהקובץ `mocks/products.json`.
- כאשר יהיו פרטי חיבור אמיתיים ל-Wix, משנים ל-`wix` **בקובץ הזה בלבד** -
  שאר הקוד לא צריך להשתנות, כי כל הקריאה למוצרים עוברת דרך `services/productService.ts`.

## איך מריצים את זה (להעברה למפתח/ת)

```bash
npm install
cp .env.example .env
npm run dev
```

האתר ייפתח בכתובת `http://localhost:3000` ויציג את 21 המוצרים מתוך `mocks/products.json` - זו הוכחה שהשלד עובד מקצה לקצה.

## מה הלאה

- שלב ו': עגלה ו-Checkout
